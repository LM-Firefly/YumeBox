为保持界面简洁，App 内的设置项大多只显示标题，详细说明集中放在本页。各小节附带「在 App 中打开」按钮，点击即可通过 `yumebox://` 深度链接直接跳转到对应界面。

<Note>
「在 App 中打开」依赖自定义协议 `yumebox://`，仅在已安装 YumeBox 的设备上有效。若点击无反应，可复制链接到浏览器地址栏打开，或在 App 内手动定位。
</Note>

## 首页手势

默认的 Moe 首页支持两种手势，无需进入设置即可操作壁纸与侧栏：

<CardGroup cols={2}>
  <Card title="单击主图" icon="image">
    单击首页顶部的壁纸图片，会直接打开系统相册选择器，选图后进入裁剪界面调整显示范围，确认即更换首页壁纸。
  </Card>
  <Card title="双击主图" icon="panel-left">
    双击壁纸图片，展开或收起左侧信息侧栏（运行时长 / 时钟、快捷入口等）。
  </Card>
</CardGroup>

因此原先「首页侧栏默认展开」开关与「首页壁纸」设置项已从设置中移除——这两项行为现在完全由首页手势接管。

<Card title="在 App 中打开首页" icon="house" href="yumebox://page/home">
  `yumebox://page/home`
</Card>

## 应用设置详解

<Card title="在 App 中打开应用设置" icon="settings" href="yumebox://screen/appsettings">
  `yumebox://screen/appsettings`
</Card>

### 行为

| 设置项 | 说明 |
| --- | --- |
| **自动启动** | 应用启动和开机时自动启动代理服务。 |
| **启动更新配置** | 启动时自动更新当前订阅配置。 |
| **坚持一个中国原则** | 自动将台湾地区旗帜及区域码显示为中国。 |

### 界面

| 设置项 | 说明 |
| --- | --- |
| **主题模式** | 选择应用的主题样式（跟随系统 / 浅色 / 深色）。 |
| **主题配色** | 点击选择主题种子色，App 据此自动推导浅色 / 深色两套配色。 |
| **反转主题前景色** | 当自动推导出的前景文字 / 图标颜色看起来怪异时，可开启此项反转前景明暗。 |
| **应用语言** | 选择应用语言（跟随系统 / 简体中文 / English）。 |
| **滑动隐藏底栏** | 内容向下滑动时自动隐藏底部浮动导航栏，向上滑动时重新显示。 |
| **背景模糊** | 为顶栏 / 底栏应用模糊背景效果。 |
| **预测返回手势** | 在 Android 14 及以上启用系统「预测返回」手势支持，返回时可预览上一个界面；开启后会重建界面。 |
| **平滑圆角** | 启用全局平滑圆角（Smooth Corner）效果。 |
| **页面缩放** | 调整 App 整体界面缩放比例，范围 80% – 120%。 |
| **经典首页** | 回退到旧版（经典）首页布局，关闭则使用默认 Moe 首页。 |

### 隐私

| 设置项 | 说明 |
| --- | --- |
| **启动时生物验证** | 每次打开 App 都需要通过生物识别或锁屏凭据验证才能进入。 |
| **禁止截图** | 禁止截图、录屏及最近任务预览。 |
| **隐藏应用图标** | 隐藏桌面图标后，可通过拨号盘输入 `*#*#0721#*#*` 重新打开 App。 |
| **隐藏后台卡片** | 启用后，App 不会出现在系统「最近任务」列表（后台卡片）中。 |

### 服务

| 设置项 | 说明 |
| --- | --- |
| **显示流量通知** | 在通知栏中显示实时流量使用情况。 |
| **单节点测试** | 点击节点卡片右侧图标，测试单个节点的延迟。 |
| **后台隐藏时释放界面** | 当 App 退到后台且界面完全不可见时，主动释放界面以节省内存，同时保留正在运行的代理服务。 |
| **电池优化无限制** | 显示当前后台是否受系统电池优化限制；受限时建议设为「无限制」，可降低后台被杀概率。 |

### 网络

| 设置项 | 说明 |
| --- | --- |
| **更新通道** | 选择检查更新时使用的 GitHub Release 通道：Stable / Pre-release / Smart。 |
| **自定义 User-Agent** | 拉取订阅时使用的 User-Agent，留空则使用默认值。 |

## 网络设置详解

<Card title="在 App 中打开网络设置" icon="network" href="yumebox://screen/network">
  `yumebox://screen/network`
</Card>

### 代理模式

| 设置项 | 说明 |
| --- | --- |
| **路由系统流量** | 选择用于接管系统流量的代理模式：HTTP 系统代理 / VPN 模式 / Root TUN。 |

### 服务配置（VPN 模式）

| 设置项 | 说明 |
| --- | --- |
| **绕过私有网络** | 绕过私有网络和本地地址，不经代理。 |
| **DNS 劫持** | 将所有 DNS 请求重定向到 YumeBox 处理。 |
| **允许应用绕过** | 允许其他应用通过自身设置绕过本 VPN。 |
| **运行 IPv6** | 允许通过 VPN 路由 IPv6 流量。 |
| **VPN 系统代理** | 仅在 VPN 模式下，为未走 TUN 的应用设置 HTTP 代理。 |

### 服务配置（Root TUN）

| 设置项 | 说明 |
| --- | --- |
| **接口名称** | RootTun 创建的虚拟网卡名（显示当前配置值）。 |
| **MTU** | RootTun 链路的最大传输单元（显示当前配置值）。 |
| **自动路由** | 自动添加转发所需的路由。 |
| **严格路由** | 仅允许命中的流量走 RootTun。 |
| **自动重定向** | 自动启用 RootTun 所需的重定向规则。 |
| **DNS 模式** | 选择 RedirHost 或 FakeIP。 |
| **FakeIP 地址段** | 仅在 FakeIP 模式下生效（显示当前配置值）。 |

### 访问控制

| 设置项 | 说明 |
| --- | --- |
| **管理访问控制列表** | 为应用和域名配置访问控制规则。 |

## 深度链接 yumebox://

可在网页、快捷方式或其他 App 中使用以下链接打开 YumeBox 并定位到指定界面。

### 主页面

| 链接 | 目标 |
| --- | --- |
| `yumebox://page/home` | 首页 |
| `yumebox://page/proxy` | 代理 |
| `yumebox://page/profiles` | 配置 |
| `yumebox://page/settings` | 设置 |

### 二级界面

| 链接 | 目标 |
| --- | --- |
| `yumebox://screen/appsettings` | 应用设置 |
| `yumebox://screen/network` | 网络设置 |
| `yumebox://screen/access` | 访问控制 |
| `yumebox://screen/override` | 覆写 |
| `yumebox://screen/providers` | 提供者 |
| `yumebox://screen/connection` | 连接 |
| `yumebox://screen/traffic` | 流量统计 |
| `yumebox://screen/log` | 日志 |
| `yumebox://screen/about` | 关于 |

<Note>
未识别的 host / 路径会被忽略，App 正常停留在当前界面。
</Note>
